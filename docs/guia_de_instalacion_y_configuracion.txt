# Guía de Instalación y Configuración

Esta guía proporciona instrucciones detalladas para configurar y ejecutar el sistema de gestión de documentos en entornos de desarrollo y producción.

## Requisitos Previos

Antes de comenzar, asegúrese de tener instalado lo siguiente:

- Node.js (v18 o superior)
- Yarn o npm
- Git

## Instalación

### 1. Clonar el Repositorio

Clone el repositorio desde el sistema de control de versiones:

````bash
git clone [URL_DEL_REPOSITORIO] frontend-document-management
cd frontend-document-management
````

### 2. Instalar Dependencias

Instale todas las dependencias del proyecto usando yarn o npm:

````bash
# Usando Yarn (recomendado)
yarn install

# O usando npm
npm install
````

### 3. Configurar Variables de Entorno

`Cree un archivo .env en la raíz del proyecto basado en el archivo .env.example` (si existe):

````bash
cp .env.example .env
````

Edite el archivo .env con sus propias configuraciones:

````
# URLs y puntos finales
NEXT_PUBLIC_API_URL=http://localhost:3000/api

# Configuración de autenticación
NEXTAUTH_URL=http://localhost:3000
NEXTAUTH_SECRET=your-secret-key

# Otras configuraciones específicas del proyecto
# ...
````

### 4. Configurar Prisma (si aplica)

Si el proyecto utiliza Prisma ORM, genere los clientes de Prisma:

````bash
npx prisma generate
````

## Ejecución del Proyecto

### Entorno de Desarrollo

Para ejecutar el proyecto en modo de desarrollo:

````bash
# Usando Yarn
yarn dev

# O usando npm
npm run dev
````

`La aplicación estará disponible en http://localhost:3000`.

### Construcción para Producción

Para construir el proyecto para producción:

````bash
# Generar cliente Prisma y construir
yarn generate_build

# O usando npm
npm run generate_build
````

### Ejecutar en Producción

Para iniciar el servidor en modo producción:

````bash
# Usando Yarn
yarn start

# O usando npm
npm run start
````

## Uso con Docker

El proyecto incluye un Dockerfile para facilitar la contenerización:

### Construir la Imagen

````bash
docker build -t frontend-document-management .
````

### Ejecutar el Contenedor

````bash
docker run -p 3000:3000 -e NODE_ENV=production frontend-document-management
````

## Verificación de la Instalación

Para verificar que todo está funcionando correctamente:

1. `Acceda a http://localhost:3000` en su navegador.
2. Verifique que la página principal se carga correctamente.
3. Intente autenticarse (si aplica) para comprobar que el sistema de autenticación funciona.

## Solución de Problemas Comunes

Si encuentra problemas durante la instalación, consulte los siguientes consejos:

### Error de Módulos no Encontrados

Si recibe errores sobre módulos no encontrados, intente limpiar la caché y reinstalar dependencias:

````bash
rm -rf node_modules
yarn cache clean
yarn install
````

### Errores de Prisma

Si hay errores relacionados con Prisma, asegúrese de que el cliente está correctamente generado:

````bash
npx prisma generate
````

Para otros problemas, consulte la sección de problemas en el repositorio del proyecto o contacte al equipo de desarrollo.
